## Архітектура (JS-first, без фреймворків і без білда)

`index.html` — лише оболонка (`<div id="app">` + мета/OG-теги + один
`<script type="module">`). Уся сторінка рендериться клієнтським JS у
`assets/js/`:

- **`content.js`** — ЩО показувати. Вбудований контент і **резерв** (fallback).
  Двомовний текст — `{ uk, en }`. Логіки немає. Формати типів — у коментарі
  на початку файлу.
- **`config.js`** — `SHEET_ID`/`SHEET_GID` Google-таблиці + розміри картинок
  `IMG`. Порожній `SHEET_ID` = таблиця вимкнена, сайт на `content.js`.
- **`sheet.js`** — вантажить контент із Google-таблиці (gviz CSV) + власний
  CSV-парсер + збірка секцій. Повертає `null` за будь-якої помилки/порожнечі
  → сайт лишається на дефолті. **DOM-free.**
- **`render.js`** — ЯК виглядає. Чисті функції `(дані, мова) → HTML-рядок`;
  `renderPage(lang, data)` приймає контент параметром. `sectionRenderers`
  диспетчеризує за `section.type` (`statement`/`split`/`gallery`/`contact`).
  Хелпер `imageUrl()` розрізняє Drive-лінк (→ Google-CDN) і шлях у репо. Немає
  стану й подій. **DOM-free** — тестується в Node.
- **`app.js`** — КОЛИ що стається. Малює дефолт одразу, потім підміняє
  контентом із таблиці; стан мови (localStorage), лайтбокс, події через
  делегування на `document` (переживають ререндер).

Джерело контенту: таблиця, якщо доступна; інакше `content.js`. Деталі
редагування через таблицю — у `SHEET.md`. `sheet-template.csv` — знімок
`content.js` у форматі таблиці; при зміні дефолту перегенеруй його.

`assets/style.css` — звичайний CSS; `render.js` навмисно емітить ті самі
класи, тому стилі правити при зміні контенту не треба.

## Правила редагування

- **95% правок — тільки в `content.js`.** Додати роботу = покласти JPEG у
  `assets/img/` і дописати об'єкт у масив `works`. Додати серію = дописати
  секцію типу `gallery` у `content.sections` (порядок у масиві = порядок на
  сторінці).
- Новий **тип секції** → функція в `sectionRenderers` (`render.js`) + новий
  `type` у `content.js` + за потреби стилі в `style.css`.
- Зберігай **обидві мови** (`uk` + `en`) при зміні тексту. Якщо `en` немає,
  `t()` падає назад на `uk` — не ламається, але краще додавати переклад.
- CSS-класи, які генерує `render.js`, мають збігатися з `style.css` — не
  перейменовуй в одному місці, не оновивши інше.

## Перевірка змін

Рендер DOM-free, тому спершу — швидкий smoke-тест у Node:

```bash
node -e "import('./assets/js/render.js').then(m => console.log(m.renderPage('uk').length, m.renderPage('en').length))"
```

Далі — у браузері (ES-модулі **не працюють** з `file://`, потрібен сервер):

```bash
python3 -m http.server 8000   # → http://localhost:8000
```

## Специфіка

- **Відео/PDF з Google Drive** відкриваються в лайтбоксі («режим
  кінотеатру»). Лінк несе `data-preview` з адресою `…/file/d/<ID>/preview`;
  `app.js` вантажить її в iframe лише при кліку і вивантажує при закритті.
  Файли на Drive мають бути «Усі, хто має посилання»; оновлювати їх треба
  через «Керування версіями» (щоб `<ID>` не змінився), не перезаливати.
- **Контактна форма** — FormSubmit.co на arrudenk@gmail.com. Після
  відправки редірект на `?sent=1#contact` → банер «Дякуємо». Перше
  повідомлення тригерить лист-активацію FormSubmit (потрібен клік).
- **`portfolio.pdf`** — джерело контенту, навмисно в `.gitignore`
  (не комітити). Також ігноруються `.idea/`, `.DS_Store`, `.claude/`.
- Сайт вимагає JS у клієнта; SEO/прев'ю тримаються на мета/OG-тегах у
  `<head>` статичного `index.html`.

## Продуктивність

Рендер — одноразова робота на мілісекунди (при завантаженні + при зміні
мови), нічого не крутиться постійно. Тримай цю властивість: не додавай
роботу в гарячі шляхи (скрол, resize) без потреби; лайтбокс-iframe вантаж
лише за дії користувача.
