# Редагування сайту через Google-таблицю

Сайт бере контент (тексти, картинки, лінки на відео/PDF) із Google-таблиці.
Редагуєш таблицю — оновлюється сайт, у код лізти не треба.

**Запобіжник:** якщо таблиця недоступна, порожня чи випадково зламана — сайт
показує вбудовану резервну версію (`assets/js/content.js`) і не «падає».
Тобто зіпсувати сайт таблицею неможливо: у найгіршому разі побачиш стару
версію, поки не виправиш таблицю.

Зміни зʼявляються на сайті за 1–2 хвилини (Google трохи кешує) — просто онови
сторінку.

---

## Одноразове налаштування (вже майже все зроблено)

Таблиця вже підключена в коді (`assets/js/config.js` → `SHEET_ID`). Лишилось
двічі клікнути в самій таблиці:

### 1. Залити початковий вміст
У репозиторії є файл `sheet-template.csv` — це весь поточний сайт у форматі
таблиці. Заливаємо його в таблицю:

1. Відкрий таблицю → меню **Файл → Імпортувати**.
2. Вкладка **Завантажити** → перетягни `sheet-template.csv`.
3. Місце імпорту: **«Замінити поточний аркуш»**.
4. **Імпортувати дані**.

Тепер таблиця містить увесь сайт, а сайт читає її з першого аркуша (gid=0).

### 2. Відкрити доступ на читання
**Надати доступ** (кнопка вгорі) → «Загальний доступ» → **Усі, хто має
посилання** → роль **Читач** → Готово. Без цього сайт таблицю не прочитає.

Готово. Далі — лише редагування.

## Як влаштована таблиця

Один рядок = один блок сайту. **Порядок рядків = порядок на сторінці**
(щоб посунути блок — перетягни рядок за номер зліва).

Колонка **Тип** каже, що це за рядок:

| Тип | Що це |
|-----|-------|
| `section` | Початок секції (розділу). |
| `group` | Підгрупа картинок усередині секції (напр. «Ескізи»). |
| `work` | Одна робота: картинка + назва + матеріали. |

Колонка **Тип-секції** (лише для рядків `section`):
- `statement` — Artist Statement (відкривається кнопкою **about** зліва
  зверху);
- `split` — текст + велика картинка збоку (проєкт у списку «дослідження»).
  Можна додати **кілька відео й/або кілька зображень**: після рядка секції
  дописуй рядки `work` — для відео заповни **Лінк-URL** = шлях до `.mp4`
  (Тип-лінка = `video`); для зображення заповни **Зображення**. Кілька відео
  запускаються по черзі (перше — одразу, решта при скролі/наведенні); зайві
  зображення показуються під головним. Головна картинка секції — у її
  власному рядку (колонка **Зображення**);
- `split-reverse` — те саме, але картинка з іншого боку;
- `gallery` — заголовок + сітка робіт (теж проєкт у «дослідженнях»);
- `header` — квадратний слайдер фото на головній (авто-гортання + крапки).
  Після рядка `section/header` додай рядки `work` — по одному на фото,
  заповнюючи лише колонку **Зображення** (шлях у репо `assets/img/…` або
  лінк на Drive). Порядок рядків = порядок слайдів. Прибрати фото — видали
  його рядок.

Порядок секцій `split`/`gallery` у таблиці = порядок проєктів у випадному
списку «дослідження». Сторінка «Записки злочинів проти тварин» збирається
з PDF у репозиторії (не з таблиці).

Колонка **Сітка** (скільки картинок у ряд): `grid-1`, `grid-2`, `grid-3`,
`grid-4`, `grid-sketches` (дрібна сітка для ескізів).

Двомовність: колонки «(укр)» і «(eng)» йдуть парами. Якщо англійську
лишити порожньою — підставиться українська.

Абзаци в текстових полях розділяй **порожнім рядком** усередині комірки
(Alt+Enter двічі).

## Найчастіші правки

### Змінити текст
Знайди потрібний рядок, відредагуй комірку «Текст (укр)» / «Текст (eng)».

### Додати нову роботу в серію
Додай **новий рядок** одразу під іншими роботами тієї серії:
- **Тип** = `work`
- **Зображення** = посилання на картинку з Google Drive (див. нижче)
- **Заголовок (укр)** / **(eng)** = назва + рік, напр. `«Лелека», 2025`
- **Матеріали (укр)** / **(eng)** = напр. `олія на полотні · 75×150`
- **Ціна** = напис у правому нижньому кутку картини. Одна колонка (не
  двомовна). Приклади: `€500`, `12 000 грн`. Значення **`0`** показує
  «в наявності / in stock». **Порожня** комірка — нічого не показує.

Щоб прибрати роботу — видали її рядок.

### Змінити відео або PDF
Знайди рядок секції з лінком, встав нове посилання в **Лінк-URL**,
а в **Тип-лінка** — `video` або `pdf`.
- **Відео:** можна вставляти посилання **YouTube** (звичайне, `youtu.be`
  чи `embed`) або Google Drive — сайт сам зробить правильний плеєр.
  Для відео радимо YouTube: надійніше грає на телефоні.
- **PDF:** посилання на файл Google Drive.

## Картинки з Google Drive

1. Завантаж картинку в Google Drive.
2. На файлі → **Надати доступ** → **Усі, хто має посилання** → **Читач**.
   ⚠️ Без цього картинка не покажеться.
3. **Копіювати посилання** і встав його в колонку **Зображення**.

Посилання виглядає так:
`https://drive.google.com/file/d/1AbC…XyZ/view?usp=sharing` — вставляй його
цілком, сайт сам візьме звідти картинку в гарній якості.

**Порада:** тримай усі картинки сайту в одній Drive-папці, якій один раз
відкрито доступ «за посиланням» — тоді кожен новий файл у ній уже доступний.

**Застереження:** Google іноді обмежує показ картинок прямо з Drive (якщо їх
дуже часто відкривають). Якщо якась картинка перестала показуватись — перевір
доступ до файлу; за потреби найважливіші фото можна покласти прямо в
репозиторій (тоді в колонці «Зображення» пишеться `assets/img/назва.jpg`).

## Для розробника

- Підключення: `assets/js/config.js` — `SHEET_ID`, `SHEET_GID`, розміри
  картинок `IMG`. Порожній `SHEET_ID` вимикає таблицю (сайт на дефолті).
- Завантаження й розбір: `assets/js/sheet.js` (CSV через gviz + власний
  парсер + збірка секцій). Повертає `null` за будь-якої проблеми → фолбек.
- `sheet-template.csv` — актуальний знімок дефолтного контенту; згенерований
  з `content.js`. Якщо міняєш дефолт — перегенеруй шаблон, щоб вони збігались.
