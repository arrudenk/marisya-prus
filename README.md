# Портфоліо — Прус Марися

Односторінковий сайт-портфоліо: https://arrudenk.github.io/marisya-prus/

Без фреймворків і без білда — чистий JavaScript (ES-модулі), який рендерить
усю сторінку з даних. Хостинг: GitHub Pages, деплой = звичайний `git push`.

## Архітектура

HTML тут — лише оболонка. Вся сторінка будується в JS із трьох шарів:

```
index.html            оболонка: <head> з мета-тегами + <div id="app"> + <script type="module">
assets/style.css      усі стилі (звичайний CSS, класи ті самі, що генерує render.js)
assets/js/
  content.js          ЩО показувати — весь контент як дані (uk/en поруч)
  render.js           ЯК виглядає — чисті функції (дані, мова) → HTML-рядок
  app.js              КОЛИ що стається — стан (мова), події, лайтбокс
assets/img/           оптимізовані JPEG (≤1600px)
```

Потік: `app.js` бере поточну мову → викликає `renderPage(lang)` з `render.js` →
той проходить `content.sections` і для кожної секції викликає рендерер за її
`type` → результат вставляється в `#app` через `innerHTML`. Рендер відбувається
двічі за сесію максимум: при завантаженні і при перемиканні мови. Це
одноразова робота на мілісекунди — нічого не перераховується постійно.

Всі обробники подій повішені на `document` (делегування), тому переживають
будь-який ререндер — нічого не треба перепідписувати.

## Як щось змінити

**Правило: 95% правок — тільки в `content.js`.** Він без логіки, це просто
великий об'єкт. Формати типів описані в коментарі на початку файлу.

### Змінити текст
Знайди рядок у `content.js`, поправ `uk` і `en` версії. Все.

### Додати роботу в існуючу серію
Поклади оптимізований JPEG в `assets/img/` і додай об'єкт у `works`
потрібної групи:

```js
{
  src: 'assets/img/nova-robota.jpg',
  alt: 'Назва роботи',
  title:     { uk: '«Назва», 2026', en: '“Title”, 2026' },
  materials: { uk: 'олія на полотні · 100×80 см', en: 'oil on canvas · 100×80 cm' },
},
```

Якщо `title` не вказати — картинка буде без підпису (як ескізи).

### Додати нову серію (секцію)
Додай у масив `content.sections` об'єкт типу `gallery` (поряд є готові
приклади). `grid` вибирає розкладку: `grid-1` (одна велика), `grid-2`,
`grid-3`, `grid-4` (2×2), `grid-sketches` (дрібна сітка 5 в ряд).
Порядок секцій на сторінці = порядок у масиві.

### Змінити контакти / Instagram / форму
Об'єкти `artist` і `form` угорі `content.js`.

### Новий тип секції
Ось тут уже `render.js`: додай функцію в `sectionRenderers` і використовуй
її `type` у `content.js`. Нові стилі — в `style.css`.

## Відео та PDF (Google Drive)

Лінки «Відеозапис роботи» і «Електронна версія щоденника» відкриваються
в лайтбоксі (затемнення + по центру, як фото). Механіка: в лінка є
`data-preview` з Drive-адресою виду
`https://drive.google.com/file/d/<FILE_ID>/preview`, `app.js` перехоплює
клік і вантажить її в iframe лайтбокса. iframe вантажиться лише в момент
кліку і вивантажується при закритті (зупиняє відтворення, нічого не їсть).

Важливо для файлів на Drive:
- доступ має бути **«Усі, хто має посилання»**, інакше буде помилка доступу;
- щоб оновити відео/PDF, **не видаляй файл** — заміни вміст через
  «Керування версіями» (три крапки на файлі). Так `FILE_ID` лишається тим
  самим і лінки на сайті не ламаються.

## Мови (uk/en)

Двомовний текст — об'єкт `{ uk: '…', en: '…' }`. Функція `t(field, lang)`
у `render.js` дістає потрібну мову; якщо `en` нема — покаже українську,
тому нічого не зламається, якщо переклад додаси пізніше. Вибір мови
зберігається в `localStorage` під ключем `lang`.

## Локальний запуск

ES-модулі не працюють з `file://` (браузер заблокує) — потрібен будь-який
статичний сервер:

```bash
cd artwebpage
python3 -m http.server 8000
# → http://localhost:8000
```

## Деплой

GitHub Pages роздає гілку `main` як є:

```bash
git add -A
git commit -m "опис зміни"
git push
```

Сайт оновиться за ~1–2 хвилини.

## Контактна форма

FormSubmit.co, надсилає на arrudenk@gmail.com (`form.action` у `content.js`).
Після відправки FormSubmit редіректить на `?sent=1#contact` — `app.js`
бачить цей параметр і показує банер «Дякуємо». Перше повідомлення з форми
тригерить лист-активацію FormSubmit — треба клікнути підтвердження.
