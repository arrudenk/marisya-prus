# Промпт для Fable: редизайн сайту-портфоліо Marysia Prus

Ти реалізуєш повний редизайн односторінкового сайту-портфоліо художниці.
Працюй **у цій гілці** (`redesign-fable`), не чіпай `main`/живий сайт. Нижче —
вичерпна специфікація з підтвердженими рішеннями. Спілкування з користувачем —
**українською** (він знає JavaScript, але не HTML/CSS — пояснюй у термінах JS).

## Архітектура (зберегти підхід — JS-first, без білда)
`index.html` — оболонка (`<div id="app">` + мета/OG + один `<script type="module">`).
Рендер клієнтським JS у `assets/js/`:
- `content.js` — дані + резерв (fallback), двомовний текст `{ uk, en }`.
- `config.js` — `SHEET_ID` / `SHEET_GID` таблиці + `IMG` (розміри картинок).
- `sheet.js` — вантажить контент із Google-таблиці (gviz CSV) + парсер + `buildSections`. DOM-free; повертає `null` за помилки → сайт на fallback.
- `render.js` — чисті функції `(дані, мова) → HTML-рядок`, `renderPage(lang, data)`. DOM-free, тестується в Node.
- `app.js` — стан (мова в localStorage), події через **делегування на `document`** (переживають ререндер), таймери, lightbox, hash-навігація.
- `style.css` — звичайний CSS; класи з render.js мусять збігатися.
Джерело контенту: таблиця, якщо доступна; інакше `content.js`. Деплой — GitHub Pages.

**Наявні патерни для повторного використання:**
- Hash-навігація накладок: лінк `#p/slug` → `hashchange` → `syncProjectFromHash`.
- Розгортання за висотою: `grid-template-rows: 0fr→1fr` (див. `.nav-dropdown`).
- Блокування фону + збереження скролу: `updateBackgroundLock()` (overflow:hidden + `scrollTo`).
- `imageUrl()` (Drive-лінк→CDN або шлях у репо), `embedUrl()` (YouTube/Drive→iframe).
- Каруселька: `setInterval` + крапки + pointer-драг, пауза поза екраном (IntersectionObserver) — реалізовано (`setupHeaderCarousel`, класи `.fc-*`).

## Підтверджені рішення (від користувача)
1. **Шрифт:** IBM Plex Mono (Google Fonts, підмножини `latin` + `cyrillic`) — скрізь, і текст, і заголовки.
2. **Сторінка «Записки»:** PDF → картинки (по сторінці), кастомний рідер.
3. **Напрям «Записок»:** сторінка 1 PDF — **внизу**, гортаєш **знизу вгору** у правильному порядку (макет = реверс PDF: остання сторінка зверху, перша знизу; старт скролу — біля низу).
4. **Таблиця:** контент редизайну — з **нової вкладки** тієї ж таблиці (новий `gid`); стара вкладка `gid=0` (живий сайт) недоторкана. У цій гілці `config.SHEET_GID` = новий gid (користувач дасть/створить вкладку).

## Макет і поведінка

### Хедер — зафіксований зверху, завжди поверх контенту
Три частини: **зліва `about`** · **центр — ім'я** · **справа `дослідження`** (випадаючий список).
- `about` → відкриває **Artist Statement**, що **розгортається донизу на весь екран** (анімація як у випадаючого списку, зверху вниз), інлайн, світла тема, **не** кінотеатр. Хедер і футер лишаються поверх.
- `дослідження` → випадає список **проєктів** (секції `split`/`gallery`). Клік по проєкту → його сторінка **відкривається прямо на сайті** (розгортається на весь екран, світла тема, не кінотеатр).

### Головна сторінка — мінімалізм
Лише **ім'я + квадратний слайдер фото** одразу під ім'ям. **Плитки/мозаїку прибрати повністю.**
- Слайдер **1:1 (квадрат)**, авто-перемикання + клікабельні крапки + свайп/драг (взяти наявний `.fc-*`, змінити `aspect-ratio` на `1/1`).
- Фото: `assets/img/header1.jpg`, `header2.jpg`, `hero.jpg` (hero не квадратний → `object-fit: cover`).

### Футер — зафіксований знизу, завжди поверх
Три частини: **зліва мова (укр/en)** · **центр — `Записки злочинів проти тварин`** · **справа `contacts`**.
- `contacts` → **режим кінотеатру** (затемнений lightbox): **маленьке віконце лише з поштою** (email як `mailto`), **без форми відправки**.
- `Записки…` → спец-сторінка (нижче).

### Сторінка «Записки злочинів проти тварин» (спец-режим)
Клік по центру футера:
1. Увесь сайт **плавно перефарбовується в чорний** (CSS-перехід у темну тему).
2. Знизу вгору **розгортається повноекранна сторінка**.
3. Контент — **PDF `Записки злочинів проти тварин.pdf` → картинки** (по сторінці). Викладка **реверсна**: остання сторінка зверху, перша — знизу; при відкритті `scrollTop = scrollHeight` (видно 1-у сторінку), читач гортає **вгору**, порядок читання правильний.
4. Темна тема; закриття повертає на сайт (перефарбування назад). Бажано керувати hash-ом (`#zapysky`) для кнопки «назад».
- Конвертація: `pdftoppm -jpeg -r 150 "Записки злочинів проти тварин.pdf" assets/img/zapysky/p` (або ImageMagick) → оптимізувати (~1400px, JPEG ~80%). Викласти `flex-direction: column-reverse` або reverse масиву; `loading="lazy"`.
- **`.gitignore` += сам PDF** (як `portfolio.pdf`); у репо — лише картинки.

### Спільне для інлайн-сторінок (about + проєкти)
Відкриваються **прямо на сайті** (світла тема, не кінотеатр), **розгортаються донизу на весь екран** з анімацією; хедер і футер лишаються зафіксованими поверх. Закриття зберігає позицію скролу головної (`updateBackgroundLock`). Бажано через hash (`#p/slug`).

## Таблиця (нова вкладка, новий gid)
Типи секцій у `sheet.js`:
- `header` — фото слайдера головної (рядки `work` → «Зображення»): header1, header2, hero.
- `statement` — Artist Statement (для `about`).
- `split` / `gallery` — проєкти (список «дослідження»).
- `contact` — email для віконця.
Прибрати непотрібне для головної (мозаїчний `divider`, «feature»-плитки) — плиток немає. «Записки» — з PDF, не з таблиці. Оновити `sheet-template.csv` (знімок нової вкладки) і `SHEET.md`.

## Шрифт
`index.html`: `https://fonts.googleapis.com/css2?family=IBM+Plex+Mono:wght@400;500;600&display=swap` (переконатись, що тягне кирилицю). У `style.css` замінити всі `font-family` (body, `h1,h2,h3`, топбар/меню/ціни) на `'IBM Plex Mono', ui-monospace, monospace`.

## Продуктивність (обов'язково)
Рендер — разова робота. Анімації — CSS `transform`/`opacity`/`grid-template-rows` (композитор). Таймери слайдера — пауза поза екраном/на прихованій вкладці, зняття старих при ререндері. Картинки — `loading="lazy"`, оптимізовані. Нічого не крутити в гарячих шляхах (скрол/resize).

## Файли
- `index.html` — шрифт-лінк, за потреби мета.
- `assets/js/config.js` — `SHEET_GID` = новий gid.
- `assets/js/render.js` — новий каркас: `header` (about/ім'я/дослідження), `footer` (мова/Записки/contacts), головна (ім'я + квадратний слайдер), інлайн-сторінки (statement + проєкти), рідер «Записки».
- `assets/js/app.js` — «дослідження»-дропдаун, інлайн-розгортання сторінок, віконце contacts (кінотеатр без форми), рідер «Записки» + перефарбування в чорне, слайдер, hash-навігація, fixed-хедер/футер поверх.
- `assets/js/sheet.js` — скориговані типи секцій.
- `assets/js/content.js` — резерв під новий дизайн (ім'я, фото слайдера, statement, проєкти, email, список картинок «Записок»).
- `assets/style.css` — IBM Plex Mono; фіксовані хедер/футер (z-index поверх); анімації розгортання; темна тема «Записок» + перехід; квадратний слайдер.
- `assets/img/zapysky/*.jpg` — сторінки PDF; `.gitignore` += PDF.

## Перевірка (E2E)
1. `node -e "import('./assets/js/render.js').then(m=>console.log(m.renderPage('uk').length, m.renderPage('en').length))"` + `node --check assets/js/app.js assets/js/sheet.js`.
2. Браузер (локальний сервер + Chrome/puppeteer):
   - Головна: лише ім'я + квадратний слайдер (плиток немає); слайдер авто-гортає, крапки/свайп працюють.
   - `about` → statement на весь екран, світла тема, хедер/футер поверх; закриття зберігає скрол.
   - `дослідження` → список проєктів; клік → проєкт інлайн (не кінотеатр).
   - `contacts` → маленьке затемнене віконце лише з поштою (без форми).
   - `Записки` → перефарбування в чорне, сторінка виїжджає знизу; 1-а сторінка внизу, скрол угору гортає далі у правильному порядку.
   - Шрифт скрізь IBM Plex Mono (computed); хедер/футер зафіксовані.
   - Скріншоти десктоп + портрет.
3. Таблиця: `SHEET_GID` → нова вкладка; `buildSections` парсить без помилок; fallback `content.js` дає той самий вигляд.

## Відкриті питання / припущення
- Дубль імені (центр хедера vs велике ім'я на головній): за замовч. одне ім'я в центрі хедера над слайдером — узгодити візуально.
- Хедер і футер — **обидва зафіксовані, завжди поверх**. Якщо хедер має ховатися при скролі — уточнити в користувача.
- `#zapysky` через hash — бажано, не критично.
